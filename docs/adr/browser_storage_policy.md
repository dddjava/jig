# ブラウザストレージ（localStorage等）を使用しない

## 状況 (Context)

JIG が生成する HTML ドキュメントは `file://` プロトコルで直接開くことができます。ブラウザによっては `file://` 上での `localStorage` や `sessionStorage` へのアクセスを制限・無効化しており、動作が一貫しません。

## 決定 (Decision)

テンプレート内の JavaScript では `localStorage`・`sessionStorage` などのブラウザストレージ API を使用しない。

UI の状態（折りたたみ状態、フィルタ設定など）はページセッション内のみで保持し、リロード後は初期状態に戻る設計とする。

## 根拠・背景 (Rationale)

- `file://` プロトコルでは `localStorage` が使用できないブラウザ・設定が存在し、エラーや無音の失敗が発生しうる
- JIG ドキュメントは HTTP サーバーを介さずローカルで参照されることを主要なユースケースとしている
- ストレージ永続化がなくても機能の本質は損なわれない
