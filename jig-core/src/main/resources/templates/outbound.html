JIG / 出力インタフェース

出力インタフェース（外部へのアクセス）の構造を可視化するドキュメントです。

用語

出力ポート
アプリケーション層で定義される出力インタフェース（例: Repository インタフェース）。「何をするか」を定義し、実装の詳細には依存しません。
アダプタ
出力ポートの実装クラス（例: JPA実装、MyBatis実装）。インフラ層に配置され、具体的なアクセス手段を担います。
永続化アクセサ
データベースへの直接アクセスを担うクラス（例: MyBatis の Mapper インタフェース）。
外部アクセサ
データベース以外の外部システムへのアクセスを担うクラス（例: HTTP クライアント、メッセージキュークライアント）。
操作対象
永続化操作の対象となるリソース（例: テーブル名）。SELECT / INSERT / UPDATE / DELETE の種別とともに表示されます。
外部型
外部アクセサが呼び出す外部システムのクラスや型。外部アクセサのメソッドが参照する先です。

ダイアグラムの構造

各要素の関係と配色は以下のとおりです。

graph LR subgraph sg_port ["出力ポート"] op(["操作"]):::outbound end subgraph sg_adapter ["アダプタ"] exec(["実行"]) end subgraph sg_pacc ["永続化アクセサ"] paccMethod(["メソッド"]) end target[("操作対象")] extAcc["外部アクセサ"] extType(("外部型")) op --> exec exec --> paccMethod paccMethod --> target exec --> extAcc extAcc --> extType classDef outbound fill:#FFF0E6,stroke:#CC6600
  • 出力ポートとアダプタは「表示要素」チェックボックスで表示・非表示を切り替えられます。
  • 永続化アクセサ・外部アクセサはデフォルトで非表示です。チェックボックスで表示できます。

タブの説明

ポート
出力ポートを起点に、アダプタ・アクセサ・操作対象の関連をポートごとに表示します。
永続化
操作対象（テーブル等）を起点に、どの出力ポートからアクセスされているかを表示します。
永続化(CRUD)
出力ポートと操作対象の対応を CRUD（SELECT / INSERT / UPDATE / DELETE）のマトリクスで表示します。
外部型
外部型を起点に、どの外部アクセサ経由でアクセスされているかを表示します。
